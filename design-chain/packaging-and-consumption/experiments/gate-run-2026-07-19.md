**Result: CONFIRMED.** The consumption toolchain is gate-executed end to end. The three new consume hooks are
scoped to `^tools/consume/`, so they fire in this repo's `--all-files` run (tools/consume/ is staged) yet stay
dormant for an adopter of `templates/git-controls/` (no tools/consume/ there → the hook is skipped). The
`.pre-commit-config.yaml` change is byte-mirrored into the template (the mirror hook passed); the
collector/runner/scan scripts are NOT mirrored. Direct runs also confirmed: shellcheck clean; 13 positive +
9 deny tests pass; the scan rejects a planted `marketplace.json`, a `*.sig`, a `.claude-plugin/*.json`
**marketplace-key** manifest (DESIGN's pinned literal), and a signing-key manifest; and it **fails closed**
(rc≠0) when run outside a git work tree.

**Residual (documented, accepted under propose-only).** The no-marketplace hook is triggered on `^tools/consume/`
to stay dormant for adopters, so a *local* `git commit` that adds a `marketplace.json`/`*.sig` elsewhere without
touching tools/consume/ is not caught by the local pre-commit stage — REQ-10 is enforced by CI's
`pre-commit run --all-files` (the merge gate), not local commits. The collector treats any exit-0 deny member as
a pass, so a *gutted* deny test (neutered to `exit 0`) would pass the collector; each real deny test carries its
own return-inversion mutation, and `t_collector_fails_closed` now exercises the collector's failing-member and
missing-member branches + pins the enumeration at 9 — a neutered deny test remains a review-caught concern.
